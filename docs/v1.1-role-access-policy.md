# CaseFlow Books Role Access Policy

Status: active from `D35-T01`

## Roles

| Role | Purpose |
|---|---|
| `customer` | Browse, maintain own profile, checkout, view own orders, and use guarded public tracking. |
| `staff` | Operate day-to-day bookstore workflows that do not change high-risk business settings. |
| `admin` | Operate the bookstore and manage high-risk settings or future privileged controls. |

## Current Permissions

| Permission | Customer | Staff | Admin |
|---|---:|---:|---:|
| `catalog:manage` | No | Yes | Yes |
| `inventory:adjust` | No | Yes | Yes |
| `orders:read` | No | Yes | Yes |
| `orders:update-status` | No | Yes | Yes |
| `promotions:manage` | No | No | Yes |
| `settings:manage` | No | No | Yes |

Current protected endpoints:

- `GET /api/admin/dashboard`: requires `orders:read`.
- `GET /api/admin/exports/orders`: requires `orders:read` and returns an
  operations CSV without customer email, phone, full address, customer id, or
  internal notes.
- `GET /api/admin/orders`: requires `orders:read`.
- `PATCH /api/admin/orders/[id]`: requires `orders:update-status` and updates
  order status, payment status, shipping status, and internal notes.
- `GET /api/admin/books/editions`: requires `catalog:manage`.
- `POST /api/admin/books/editions`: requires `catalog:manage`.
- `PATCH /api/admin/books/editions/[id]`: requires `catalog:manage`.
- `GET /api/admin/inventory`: requires `inventory:adjust`.
- `POST /api/admin/inventory/adjustments`: requires `inventory:adjust`.
- `GET /api/admin/promotions`: requires `promotions:manage`.
- `POST /api/admin/promotions`: requires `promotions:manage`.
- `PATCH /api/admin/promotions/[id]`: requires `promotions:manage`.
- `GET /api/admin/customers`: requires `orders:read`.
- `GET /api/admin/settings`: requires `settings:manage`.

## Boundaries

- UI visibility is not an authorization boundary.
- Staff and admin checks must run on server pages and Route Handlers.
- Customers must receive `FORBIDDEN` for admin/staff APIs.
- Anonymous callers must receive `UNAUTHORIZED` for admin/staff APIs.
- Staff may access operational order screens, but staff must not manage
  high-risk settings unless a later roadmap task or ADR explicitly allows it.
- Browser payloads must never choose role, permission, order status authority,
  price, stock, tax, fee, or total.
- Order, payment, and shipping status transitions must be enforced on the
  server. UI select options are only guidance, not an authorization boundary.

## Verification

`caseflow-store/scripts/verify-staff-role-access.ts` verifies anonymous,
customer, staff, and admin outcomes for direct protected APIs and staff/admin
operations page access.
