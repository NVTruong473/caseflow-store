# CaseFlow Books v1.1 Data/Domain Freeze

- Status: Accepted
- Date: 2026-07-16
- Roadmap gate: `D25-T03 - Accept Data/Domain Freeze`

## Scope

This freeze closes the data, domain, seed, repository, and public catalog API
foundation for the CaseFlow Books `v1.1` storefront work that begins on Day 26.
It is not a storefront, checkout, account, payment, operations, SEO, assistant,
or deployment freeze.

The frozen foundation includes:

- Book domain contracts in `caseflow-store/src/types/domain.ts`.
- Book Zod validation schemas in `caseflow-store/src/lib/validation/domain.ts`.
- Book query validation in `caseflow-store/src/lib/validation/books.ts`.
- Supabase book schema migration:
  `caseflow-store/supabase/migrations/0006_caseflow_books_schema_draft.sql`.
- Safe public-domain/classic seed dataset:
  `caseflow-store/src/data/books/seed.ts`.
- Safe cover placeholder strategy and asset:
  `caseflow-store/public/images/books/placeholders/book-cover-placeholder.svg`.
- Book row mappers:
  `caseflow-store/src/lib/supabase/book-mappers.ts`.
- Book catalog repository:
  `caseflow-store/src/lib/repositories/supabase-books.ts`.
- Public catalog API DTO serializer:
  `caseflow-store/src/lib/api/book-catalog.ts`.
- Public catalog routes:
  - `GET /api/categories`
  - `GET /api/products`
  - `GET /api/products/[slug]`
- Stable API envelope:
  `{ data, error, meta }`.
- Stable book not-found error code:
  `BOOK_EDITION_NOT_FOUND`.

## Frozen Decisions

- CaseFlow Books remains a focused bookstore and small-business operations demo,
  not a marketplace.
- VND remains the authoritative stored currency.
- Book catalog rows are separate from legacy phone-accessory rows.
- Legacy phone-accessory tables remain in place until a later contract phase
  removes or archives them.
- Public catalog APIs return book category and book edition DTOs after Day 25.
- The old phone-accessory repository remains available only for legacy UI/cart
  flows that have not yet been migrated.
- Book summaries remain internally written; commercial blurbs, reviews, and
  protected excerpts are not copied.
- Commercial book covers are not used without a clear allowed source.
- ISBNs remain `null` unless verified identifiers are intentionally added later.
- Phone verification and real payment processing are not claimed.

## Allowed After This Freeze

- Rebrand UI from CaseFlow Store to CaseFlow Books.
- Build bilingual UI around the frozen book domain and DTOs.
- Build book homepage, listing, filters, details, and cart behavior against the
  frozen public catalog APIs.
- Add customer account, checkout, payment simulation, admin operations, SEO,
  assistant, and analytics work when their roadmap tasks arrive.
- Add tests, screenshots, documentation, and bug fixes.
- Add non-breaking UI-only formatting and presentation helpers.

## Requires Explicit Review Before Changing

- Book table shape, constraints, RLS, grants, or migration behavior.
- Book domain type names, field names, language/format/status enums, or money
  semantics.
- Public catalog API response envelope or error contract.
- Existing public catalog query parameter names or meanings.
- Seed licensing policy, cover-source policy, or summary/content policy.
- Currency source-of-truth rules.
- Account-gated checkout or payment-provider assumptions.

## Known Risks Before Storefront Expansion

- The public catalog API still uses the existing `/api/products` path while
  returning book edition DTOs. This is intentional for D25, but the naming is
  semantically awkward. A future task may add canonical `/api/books` routes if
  the roadmap accepts that compatibility change.
- Active storefront pages still render the legacy phone-accessory experience.
  Day 26-30 tasks own the visible UI, catalog, detail, and cart pivot.
- Existing v1.0 release E2E tests that assert old product API detail responses
  are no longer the right contract for the catalog API after D25.
- The repository filters in memory after reading the 100-edition catalog. This
  is acceptable for the current portfolio scale but not a large-catalog search
  architecture.
- All editions currently share a safe placeholder cover. This avoids copyright
  risk but is visually less realistic than a licensed or generated cover set.
- Translator rows are not seeded yet. Vietnamese editions can still be sold,
  but translator metadata should be added later if real translation editions
  are represented.
- Book order RPC/schema exists, but account-gated checkout and book order API
  wiring are intentionally scheduled for later roadmap days.
- Public API paths and legacy UI are temporarily inconsistent during the pivot.
  This should be resolved by Day 30 storefront freeze.

## Freeze Statement

As of `D25-T03`, the CaseFlow Books data/domain/API foundation is frozen for
Day 26 storefront expansion. Future tasks should build on this foundation
instead of reshaping it, unless a defect or explicitly approved roadmap change
requires reopening the data/domain contract.
