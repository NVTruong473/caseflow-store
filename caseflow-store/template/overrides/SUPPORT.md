# Support Policy

Repository access does not include customization, deployment, catalog
migration, provider setup, operations, data recovery, or a service-level
commitment.

Before buyer delivery, the written agreement must define the supported
template version, customization scope, acceptance criteria, support period,
response channel, infrastructure ownership, and excluded services.

Security findings must follow `SECURITY.md`. Do not place credentials,
personal data, buyer records, or private infrastructure details in issues.
