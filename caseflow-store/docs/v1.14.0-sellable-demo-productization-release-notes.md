# v1.14.0 - Sellable Demo Productization

Release date: 2026-07-24

## Summary

`v1.14.0` turns the CaseFlow Books reference application into a more
disciplined source-code handoff for a future bookstore buyer. It centralizes
public store identity, removes invented support contacts, adds buyer and
catalog migration contracts, and introduces an automated productization gate.

This release does not claim that the reference deployment is a legally or
operationally complete business.

## Productization

- Added typed build-time configuration for store name, short mark, bilingual
  taglines, canonical URL, support hours, optional monitored phone/email, legal
  display name, and copyright year.
- Connected header, footer, metadata, SEO, policies, notifications, customer
  copy, operations fallbacks, and demo-payment merchant identity to shared
  configuration.
- Removed invented hotline and support mailbox defaults. Unconfigured contact
  channels are omitted instead of presenting fictional business details.
- Added default and buyer-override configuration verification.
- Added a repository-wide runtime brand-coupling scan and removed all findings
  outside the explicit reference configuration, policy templates, and catalog
  source data.

## Buyer Handoff

- Added a buyer discovery questionnaire.
- Added a storefront customization guide and commercial boundary checklist.
- Added a non-destructive catalog replacement contract with mapping,
  provenance, backup, dry-run, reconciliation, rollback, and order-snapshot
  safeguards.
- Added a truthful buyer demo script and a reference deployment, incident, and
  rollback runbook.
- Documented that rebranding remains a reviewed build/deploy activity, not a
  multi-tenant theme builder.

## Reliability

- Reused the first 500-edition catalog snapshot for filtering instead of
  repeating the full relational Supabase read in the same request.
- Updated the catalog verifier to use the current configurable 500-edition
  baseline rather than the obsolete 100-edition assumption.
- Preserved server-side price/stock recalculation, authorization, notification,
  payment, and stable API boundaries.

## Boundaries

- Payments remain simulated; Production mock-payment success is locked.
- External email/SMS delivery and real OTP providers remain disabled.
- No real bank account, carrier, warehouse, custom domain, or observability
  vendor was configured.
- The 500-edition catalog is reference data. A buyer must replace or approve
  products, prices, stock, merchandising copy, media, provenance, and rights.
- Buyer-specific legal terms, tax treatment, privacy commitments, support
  channels, integrations, credentials, and third-party costs require a
  separate discovery, implementation, security review, and acceptance cycle.
