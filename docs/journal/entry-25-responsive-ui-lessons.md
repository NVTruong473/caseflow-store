# Entry 25 - Responsive UI Lessons

Type: retrospective note, not an additional development day.

## What Held Up

Testing at 375px, 768px, 1024px, and 1440px caught problems that a desktop-only workflow would miss. The admin tablet overflow fix is the clearest example.

The project benefited from stable product image ratios, constrained content widths, mobile menu states, and separate mobile admin cards. These choices made the app feel more intentional without adding flashy UI.

Loading, empty, error, and success states made the site feel more complete than a simple happy-path demo.

## What Was Risky

Responsive polish can become subjective if it is not tied to evidence. Screenshots, DOM overflow checks, and role-based interaction tests kept the review grounded.

## Portfolio Takeaway

The UI is strongest where it proves real behavior: product browsing, cart drawer interaction, checkout validation, admin status updates, and mobile recovery states.
