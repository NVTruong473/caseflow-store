# v1.11.1 - Security Dependency Patch

Date: 2026-07-22

Production URL: `https://caseflow-store.vercel.app`

## Summary

`v1.11.1` is a focused dependency security patch after the `v1.11.0`
account-security release. It does not add new user-facing features, schema
migrations, payment behavior, or authentication flows.

The patch addresses npm audit findings for the Next.js dependency tree without
using the unsafe `npm audit fix --force` downgrade path.

## Changes

- Updated `next` from `16.2.10` to `16.2.11`.
- Updated `eslint-config-next` from `16.2.10` to `16.2.11`.
- Added npm `overrides` for the Next dependency tree:
  - `postcss` -> `8.5.19`
  - `sharp` -> `0.35.3`
- Regenerated `package-lock.json`.

## Why This Was Needed

`npm audit --audit-level=high` reported:

- `sharp <0.35.0` inherited libvips vulnerabilities.
- `postcss <8.5.10` under the Next.js dependency tree.

The default npm fix path proposed a breaking downgrade to an old Next.js
version, so this release uses a narrow patch upgrade plus explicit overrides.
