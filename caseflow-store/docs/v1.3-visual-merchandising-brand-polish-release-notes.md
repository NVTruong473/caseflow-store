## What Changed

- Expanded the bookstore visual system with paper/ink, discovery, editorial,
  offer, and admin trust tokens.
- Added reusable cover-led merchandising components for local CaseFlow-created
  book cover assets.
- Polished the homepage hero, featured shelf, catalog cards, discovery
  filters, book detail hierarchy, edition comparison, and admin operations
  surfaces.
- Improved visual QA scripts so lazy-loaded covers are warmed before full-page
  screenshots and transient duplicate dev-server nodes do not create false
  strict-selector failures.
- Fixed a real staff catalog update regression where schema defaults for
  source-review fields were treated as explicit source-review edits even when
  PATCH bodies only changed operational fields.

## What Did Not Change

- No database schema migration was added.
- No production data import or destructive data operation was performed.
- No real payment, shipping carrier, SMS/OTP, or email verification provider
  was integrated.
- No external stock imagery, commercial book covers, fake ratings, fake sold
  counts, or bestseller claims were added.
- Production deployment, smoke testing, release documentation, release commit,
  and annotated `v1.3.0` tagging were completed on 2026-07-18.
- `v1.3.0` was superseded by the `v1.3.1` compact-card layout hotfix after a
  user-reported related-book card overlap defect was verified and fixed.

## Verification Summary

- V13 aggregate visual audit: passed for homepage, catalog, book detail,
  checkout/account boundary, admin dashboard, and admin catalog at mobile and
  desktop sizes.
- V13 focused verifiers: design tokens, cover merchandising components,
  homepage merchandising, catalog visual merchandising, book detail hierarchy,
  and admin visual system all passed.
- Affected regressions passed: homepage UI/sections, catalog page, catalog
  filters, V12 catalog discovery, book detail page, V12 edition comparison,
  admin dashboard, and admin catalog CRUD.
- Static gates passed: `npx tsc --noEmit --pretty false`, `npm run lint`,
  `npm run build`, and `git diff --check`.
- Production release smoke passed against `https://caseflow-store.vercel.app`
  with 100 active editions, 100 cover responses, 50 English editions, 50
  Vietnamese editions, and all public/account/admin boundary checks passing.

## Residual Notes

- v1.3 is intentionally a visual-merchandising polish phase, not a new
  commerce scope expansion.
- The project still uses project-created illustrative SVG covers instead of
  commercial publisher cover art.
- Payment and account verification remain simulated by design and should not
  be described as production payment or real OTP/email verification.
