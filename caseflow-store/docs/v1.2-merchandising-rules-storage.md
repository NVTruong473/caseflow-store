# CaseFlow Books v1.2 Merchandising Rules And Storage Contract

Status: accepted for `V12-T09` on 2026-07-17.

## Outcome

`V12-T09` defines the merchandising shelf contracts that later migration and UI
tasks must implement. It does not apply a Supabase migration and does not change
runtime storefront behavior.

The accepted outputs are:

- `src/types/merchandising.ts`
- `src/lib/validation/merchandising.ts`
- `src/lib/merchandising/shelves.ts`
- `scripts/build-v12-merchandising-rules.ts`

## Shelf Model

Approved shelf types:

- `editorial-curation`
- `category-focus`
- `language-focus`
- `promotion-focus`
- `inventory-focus`
- `paired-editions`
- `order-derived`

Approved source kinds:

- `editorial`
- `catalog-rule`
- `promotion-rule`
- `inventory-rule`
- `order-derived`

Editorial shelves use manual edition slots and are explicitly different from
catalog-rule or order-derived shelves. Promotion, inventory, language, category,
and paired-edition shelves reuse existing catalog/store data where practical.

Sales/order-derived shelves must start disabled unless a first-party query,
time window, metric, and minimum-data rule are present. The accepted fixture
keeps `sales-trending-30d` inactive with:

- query id: `first-party-orders-units-sold-v1`;
- metric: `units-sold`;
- window: `30` days;
- minimum orders: `20`.

## Permission Boundary

Merchandising mutations require `merchandising:manage`. The contract allows
admin and staff actors only when that permission is present. Anonymous,
customer, and staff-without-permission actors are denied.

Runtime admin APIs are not changed in this task. Server-side checks are required
when `V12-T15` adds admin merchandising operations.

## Minimal Storage Contract For T10

The migration in `V12-T10` should be additive and reversible. A minimal storage
shape is:

```sql
create table public.book_merchandising_shelves (
  id uuid primary key,
  slug text not null unique,
  shelf_type text not null,
  source_kind text not null,
  labels jsonb not null,
  description jsonb not null,
  inclusion_rule jsonb not null,
  starts_at timestamptz,
  ends_at timestamptz,
  is_active boolean not null default false,
  sort_order integer not null,
  min_items integer not null,
  max_items integer not null,
  fallback jsonb not null,
  required_permission text not null default 'merchandising:manage',
  created_at timestamptz not null default timezone('utc', now()),
  updated_at timestamptz not null default timezone('utc', now()),
  constraint book_merchandising_shelves_window check (
    starts_at is null or ends_at is null or starts_at < ends_at
  ),
  constraint book_merchandising_shelves_items check (
    min_items >= 0 and max_items > 0 and min_items <= max_items
  ),
  constraint book_merchandising_shelves_permission check (
    required_permission = 'merchandising:manage'
  )
);

create table public.book_merchandising_shelf_items (
  id uuid primary key,
  shelf_id uuid not null references public.book_merchandising_shelves(id)
    on delete cascade,
  edition_id uuid not null references public.book_editions(id)
    on delete restrict,
  position integer not null,
  is_active boolean not null default true,
  note text,
  created_at timestamptz not null default timezone('utc', now()),
  updated_at timestamptz not null default timezone('utc', now()),
  unique (shelf_id, edition_id),
  unique (shelf_id, position),
  constraint book_merchandising_shelf_items_position check (position > 0)
);
```

Recommended indexes:

```sql
create index book_merchandising_shelves_active_order_idx
  on public.book_merchandising_shelves (is_active, sort_order);

create index book_merchandising_shelf_items_shelf_order_idx
  on public.book_merchandising_shelf_items (shelf_id, is_active, position);
```

Recommended RLS posture:

- public storefront reads only active shelves and active shelf items;
- admin/staff mutations require server-side role checks and
  `merchandising:manage`;
- service role performs migration/upsert operations;
- customer and anonymous users cannot mutate merchandising data.
