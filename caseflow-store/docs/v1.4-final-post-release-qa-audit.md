## Notes

- A broad stale-release scan matched historical `v1.3.1` records inside agent
  history. A targeted scan over public README, app README, release docs,
  architecture, known limitations, CV bullets, and portfolio handoff found no
  stale current-release claim; current public docs point to `v1.4.0`.
- `npm audit --audit-level=high` passed. The known moderate PostCSS advisory
  remains inside the Next dependency tree; `npm audit fix --force` proposes a
  breaking downgrade path and was not applied.
- During manual visual review, the production admin login still showed one
  low-severity vendor-facing label, `Supabase Auth`. This is not a P0/P1
  release blocker because authorization remains server-owned and the route is
  an operations boundary, but it weakens the "real business" feel. A local
  source patch changes that label to business-facing, localized copy; it is
  not part of the already published `v1.4.0` tag or production deployment.
- Existing honest product boundaries still apply: payment choices are handled
  inside the checkout flow without live provider settlement, shipping is
  policy/calculation driven without carrier integration, and email/SMS
  verification providers are not connected.
