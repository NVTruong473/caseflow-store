# CaseFlow Books v1.10.0 Account-Bound Signup Voucher Release Notes

- Date: 2026-07-21
- Status: release candidate prepared; production deployment and production
  smoke evidence are recorded in the GitHub Release and `.agent` artifacts
  after deployment
- Production URL: `https://caseflow-store.vercel.app`
- Runtime scope: account-bound signup vouchers, customer-visible promotion
  surfaces, and the accepted real-cover retail UI polish already verified
  locally

## What Changed

- Added 3 fixed welcome campaigns for new customer accounts:
  `WELCOME30K`, `READMORE20K`, and `FREESHIP25K`.
- Added `customer_promotion_vouchers` for per-account voucher grants, 30-day
  expiry, reservation tokens, used timestamps, and redeemed-order relation.
- Added a server-side signup voucher repository for grant, list, reserve,
  confirm, and release operations.
- Updated customer signup and account pages so eligible customers receive and
  can see their 3 welcome codes.
- Updated checkout so customers can apply one available account code from a
  visible voucher panel while still seeing the editable promotion-code input.
- Updated order creation so the server verifies voucher ownership, expiry,
  reservation state, and one-code input before applying a discount.
- Added rollback-safe reservation behavior: if order creation fails, the
  reserved signup voucher is released instead of being burned.
- Added homepage and account CTAs inviting registration for welcome codes
  without fake countdowns, fake scarcity, or unverifiable customer counts.
- Carried forward the real-cover retail UI polish: warmer paper surfaces,
  retail-red commerce actions, stronger header/search treatment, cover-led
  shelf framing, and no-overflow render checks across home, catalog, and
  product detail pages.

## Boundaries

- No browser-owned voucher eligibility, discount amount, order total, or
  customer ownership decision was added.
- No real email, SMS, marketing automation, loyalty provider, or third-party
  coupon engine was added.
- No fake urgency, fake scarcity timer, fake user count, fake review, or fake
  sales claim was added.
- QR demo payment remains locked from production settlement.
- The release does not add real payment settlement, refunds, provider
  reconciliation, shipping-carrier integration, or real email verification.
