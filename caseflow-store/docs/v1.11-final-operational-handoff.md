## Demo Flow

### Public flow

1. Open `https://caseflow-store.vercel.app`.
2. Show homepage merchandising, language switcher, search, categories, and
   catalog density.
3. Open catalog and filter by category, language, format, price, or stock.
4. Open a product detail page and show edition comparison, VND price, stock,
   related books, and add-to-cart.
5. Open checkout while anonymous to show the account gate.
6. Use order tracking with invalid details to show protected public lookup.
7. Open admin routes while signed out to show protected access boundaries.

### Customer flow

1. Register a fresh customer email.
2. Confirm the email from the mailbox.
3. Verify the redirect lands on `https://caseflow-store.vercel.app/account`.
4. Show the 3 account-bound vouchers: `WELCOME30K`, `READMORE20K`,
   `FREESHIP25K`.
5. Complete profile readiness.
6. Add one book to cart.
7. Checkout with exactly one owned voucher.
8. Confirm the order appears in account order history.
9. Show that production QR/payment simulation endpoints remain locked.

Do not publish reusable passwords in portfolio material. Use throwaway accounts
or private credentials during live demos.

### Admin/staff flow

Use private admin/staff credentials only in a controlled environment. Show:

- dashboard metrics;
- order filtering and rejection/cancellation workflow;
- catalog and inventory operations;
- promotions/customer/settings views;
- CSV export;
- server-side access boundaries.

## Operational Controls Already In Place

- Supabase Auth URL Configuration points to the production domain.
- Redirect URLs includes `https://caseflow-store.vercel.app/account`.
- Vercel Production includes `NEXT_PUBLIC_SITE_URL`.
- Server code recalculates cart/order totals.
- Browser cart stores only edition ID and quantity.
- Customer, staff, and admin access is checked server-side.
- QR payment simulation is disabled in production.
- Sensitive payment credentials are not collected.
- Service-role operations stay server-only.

## Remaining Blockers

### `AUTH-SMTP-T02`

Custom SMTP cannot be configured until these real values exist outside the repo:

- `SUPABASE_ACCESS_TOKEN`
- `SMTP_ADMIN_EMAIL`
- `SMTP_HOST`
- `SMTP_USER`
- `SMTP_PASS`

When those values are available, run:

```bash
AUTH_SMTP_APPLY=true npm exec -- tsx scripts/configure-supabase-custom-smtp.ts
```

Then rerun strict email UAT with a fresh mailbox alias.

### Commercial readiness

Before using the app for real business traffic, add:

- real payment provider integration and reconciliation;
- SMS or phone verification;
- production observability and alerting;
- abuse/rate-limit controls;
- backup/restore drill;
- privacy retention and deletion workflows;
- licensed metadata/media pipeline.

## Recommended Portfolio Positioning

Use this wording:

> CaseFlow Books is a deployed full-stack bookstore portfolio app with a
> bilingual 500-edition catalog, account-gated checkout, server-owned totals,
> signup vouchers, email-confirmed accounts, customer order history,
> production-locked QR payment simulation, signed-in password change, and
> admin/staff operations. It is intentionally honest about its boundaries:
> payments are simulated, custom SMTP is pending, and commercial operations
> would require provider integrations and monitoring.

Avoid these claims:

- "Processes real payments."
- "Production payment QR works with real banks."
- "Business-grade email deliverability is complete."
- "Real SMS/phone verification."
- "Licensed commercial book data feed."
- "Load-tested commercial SLO."
