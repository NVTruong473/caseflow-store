## What Changed

- Added search-first desktop navigation for title, author, ISBN, or catalog
  text, submitting directly to `/catalog?q=...`.
- Added live category discovery in the desktop header using active category
  data.
- Added mobile drawer search and category links so discovery does not depend
  on homepage-only sections.
- Added a cover provenance manifest for the active 500-edition catalog:
  `assets/book-covers/sources.json`.
- Updated cover rendering to use `object-fit: contain` and explicit
  cover-updating labels for fallback art.
- Added restrained CSS motion tokens and product-card hover feedback with
  `prefers-reduced-motion` support.
- Added a reduced-motion-aware back-to-top control for long catalog and detail
  pages.
- Added a V18 browser verifier for search, category navigation, mobile menu,
  cover rendering, motion tokens, no-overflow checks, and provenance manifest
  assertions.
- Hardened Playwright cart seeding so E2E fixtures seed `localStorage` before
  React cart hydration and do not clear checkout success snapshots on later
  navigations.

## Boundaries

- This release does not add a database migration, production data mutation,
  real payment provider, shipping carrier, email/SMS provider, commercial
  cover import, external AI service, reviews, ratings, wishlist, newsletter, or
  quick-view modal.
- The 500 products remain edition variants across 50 works.
- Cover art remains local/project-generated portfolio art, not official
  publisher cover artwork.
- QR payment remains a development/sandbox demo flow and is locked from
  production settlement.
