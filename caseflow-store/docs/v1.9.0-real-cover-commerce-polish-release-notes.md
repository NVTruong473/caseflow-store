## What Changed

- Added a V19 cover-source ADR and roadmap for realistic cover assets without
  hotlinking remote images or pretending unverified commercial covers are
  licensed.
- Downloaded 49 Project Gutenberg source-work cover thumbnails into
  `public/images/books/gutenberg-covers`.
- Stripped JPEG/SVG metadata from public image assets and added a verifier for
  embedded generator/provenance metadata.
- Upserted 49 `public-domain` cover asset rows and updated 490 active edition
  cover references in Supabase production after the assets were deployed.
- Regenerated `assets/book-covers/sources.json` from the production catalog:
  500 products, 500 verified covers, 490 `public-domain-local`, 10
  `project-generated`, 0 fallback.
- Left the 10 *The Old Man and the Sea* editions on existing generated covers
  because no matching public-domain Gutenberg cover was mapped.
- Updated the homepage hero into a more commerce-focused retail floor with
  catalog search, quick category links, browse/order-tracking CTAs, live front
  table shelf, and clearer purchase signals.
- Hardened the rule-based assistant so suggested customer questions answer
  cleanly, malformed catalog questions recover, and admin/security/off-topic
  requests return the approved neutral fallback.
- Updated V18/final QA verifiers so the new accepted cover-source mix is not
  treated as missing cover art.

## Boundaries

- Fahasa automation was not used because direct automated requests returned
  anti-bot challenge/blocked responses. Future Vietnamese official cover work
  should use reviewed direct image URLs or licensed assets.
- No remote cover URL is hotlinked by the storefront.
- The Project Gutenberg source is recorded in manifests and internal docs, not
  used as a customer-facing brand claim.
- The catalog still has 500 sellable editions across 50 works; this release
  does not add 500 unique works.
- No real payment, shipping, email, SMS, review, rating, or external AI
  provider was added.
