# Security Policy

## Supported Version

Security fixes are evaluated for the latest deployed `v1.14.x` showroom
baseline. Older tags are retained as release evidence and are not maintained
independently.

## Report A Vulnerability

Use GitHub's private vulnerability reporting flow under the repository
`Security` tab. Do not disclose a suspected vulnerability in a public issue,
pull request, discussion, screenshot, or demo account.

Include:

- the affected route, release, and environment;
- reproducible steps with secrets and personal data removed;
- the expected and observed security boundary;
- the practical impact;
- a minimal proof of concept, when safe.

Do not test with another person's account or data, perform denial-of-service
testing, send real payments, or publish credentials. There is no public bug
bounty or guaranteed response-time commitment.

Buyer deployments require their own security contact, incident process,
provider review, and supported-version policy under the buyer agreement.
