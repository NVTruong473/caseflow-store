# CaseFlow Books v1.1 Roadmap

Status: accepted planning roadmap.

This roadmap governs the post-`v1.0.0` upgrade from CaseFlow Store to
CaseFlow Books. It implements the direction accepted in
[ADR-0006](adr/0006-pivot-to-caseflow-books.md).

## Non-Negotiable Guardrails

- Work one task ID at a time.
- Do not add features outside this roadmap without a new ADR or explicit user
  approval.
- Keep the Next.js modular monolith, Supabase, Vercel, Zod, and Playwright
  stack unless a new ADR supersedes it.
- Keep VND as the authoritative stored currency.
- Do not trust browser-supplied price, subtotal, stock, tax, fee, role, or
  order status.
- Do not collect card number, CVV, card expiry, or real e-wallet credentials.
- Do not claim real payment processing for simulated payment flows.
- Do not claim verified phone numbers without a real verification provider.
- Do not copy copyrighted book descriptions or cover images unless the source
  clearly permits that use.
- Use self-written summaries and safe placeholder/generated/internal cover
  assets when rights are unclear.
- Treat VAT, FX rates, and international payment fees as configurable estimates,
  not legal or tax advice.
- Every code task must run `npm run lint`; broader or shared changes must also
  run `npm run build`.
- From the affected-flow E2E phase onward, run Playwright for the changed
  customer/admin flow.

## Out Of Scope For v1.1

- Real payment gateway integration.
- Real MoMo, ZaloPay, VNPay, bank, or card credentials.
- Real SMS/OTP phone verification.
- Multi-vendor marketplace.
- Shipping-carrier integration.
- Invoice/e-invoice compliance.
- AI chatbot using external model APIs.
- Elasticsearch, Redis, queues, Kubernetes, or microservices.
- Native mobile app.
- Copying commercial book covers or publisher descriptions without permission.
