## What Changed

- Reworked customer-facing runtime copy so the storefront reads like an
  operating bookstore instead of exposing demo/internal implementation wording.
- Added token-backed visual roles for translation, academic, trust, arrival,
  and operations surfaces.
- Added a merchandising layout library for editorial feature shelves, deal
  strips, translation pairs, category spine rails, reading paths, and compact
  retail tiles.
- Upgraded homepage, catalog, product detail, checkout, account, tracking,
  assistant, footer, and admin surfaces with structurally varied layouts rather
  than recolored copies of the same card pattern.
- Added bilingual customer-facing contact, shipping, payment, returns,
  privacy, and terms pages, and linked them from footer/sitemap/robots.
- Improved admin dashboard, orders, catalog, inventory, and customer surfaces
  with operations rails, dense status panels, and mobile-friendly admin
  treatments.
- Stabilized verification scripts for API-session logins and screenshot capture
  where older Playwright helpers could create misleading local-only failures.

## What Did Not Change

- No database schema migration or production catalog import was performed.
- No real payment provider, shipping carrier, SMS/OTP, email verification
  provider, or external AI service was integrated.
- No fake reviews, ratings, sold counts, bestseller claims, testimonials, or
  urgency signals were added.
- No commercial book-cover images, publisher blurbs, protected excerpts, or
  marketplace media were imported.
- Server-owned pricing, stock, VAT, shipping, payment-fee, promotion, total,
  role, and API authorization boundaries remain unchanged.

## Residual Notes

- The app remains a commerce portfolio system with simulated payment states;
  it does not collect payment credentials or settle real transactions.
- Phone and email profile fields are required for checkout readiness but are
  not verified by provider-backed OTP/email ownership checks.
- Shipping, VAT, FX, and international payment-fee values remain configurable
  estimates.
- The active catalog uses project-created CaseFlow cover artwork. The
  user-provided Project Gutenberg, Standard Ebooks, and Vietnamese ebook/PDF
  sources are recorded as future candidates, not approved runtime cover assets
  without provenance review.
- `npm audit --audit-level=high` passes; the known moderate Next/PostCSS
  advisory remains because the available automated forced fix proposes a
  breaking downgrade path.
