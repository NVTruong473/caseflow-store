# v1.11.0 - Account Security Password Change

- Release date: 2026-07-22
- Production URL: `https://caseflow-store.vercel.app`
- Deployment: `dpl_DtUDA7pbv7ZcJYFRM5TVmsQUhThq`
- Release type: minor account-security feature

## Summary

`v1.11.0` adds a signed-in password-change flow for CaseFlow Books accounts.
Customers, staff, and admins can change their own password from the account
page after re-entering the current password.

This release does not add an admin reset-password control for other accounts.
That remains a separate support/security process and should use Supabase Auth
admin tooling or a reviewed reset-email workflow.

## Changes

- Added `PATCH /api/customer/password`.
- Added server-side Zod validation for password-change requests.
- Added `PASSWORD_UPDATE_FAILED` to the stable API error-code list.
- Added an account-page password panel before the delivery-profile form.
- Re-authenticates the current password before calling Supabase Auth
  `updateUser`.
- Rejects mismatched confirmation and same-password changes.
- Added `scripts/verify-customer-password-change.ts` for local and production
  verification.
- Updated `.env.local` locally with non-secret SMTP helper values:
  `SUPABASE_PROJECT_REF`, `SMTP_PORT=587`, and
  `SMTP_SENDER_NAME=CaseFlow Books`.

## Known Limits

- Custom SMTP is still blocked until real SMTP credentials and a Supabase
  Management API token are provided.
- This release does not implement password reset by email or admin reset for
  another account.
- Production email confirmation should still be revalidated after custom SMTP
  is configured.
