## QA Method

This audit treats the released site as a tester would: verify live production
without mutating unnecessary data, then verify full commerce/admin journeys on
a production-style local server with cleanup.

The checklist is informed by:

- ISTQB regression and acceptance-test discipline.
- OWASP Web Security Testing Guide for auth, authorization, input, and leakage
  checks.
- Lighthouse-style performance, accessibility, SEO, and best-practice gates.
- WCAG 2.2 accessibility expectations for focus, labels, and responsive use.
- Baymard e-commerce UX patterns for product discovery, cart, checkout, and
  mobile commerce friction.

## Coverage

- Storefront: homepage, catalog, catalog search/filter/pagination, product
  detail, edition comparison, add-to-cart, cart drawer, and checkout boundary.
- Customer: account boundary, account-gated checkout, validation, success
  fallback, order tracking error/success patterns through E2E coverage.
- Admin/staff: protected admin boundary, admin login, dashboard controls,
  order workflow, role access matrix, and protected API behavior.
- Content: 100 active editions, 50 English and 50 Vietnamese editions, 100
  cover responses, bilingual summaries/alt text, and no public source-review
  field leakage.
- UX: mobile and desktop screenshots, no horizontal overflow, actionable empty/
  loading/error states, visible focus, assistant guidance, and cart/checkout
  clarity.
- Security/release hygiene: protected API boundaries, stable API errors,
  cleanup, secret-like scan, stale-claim scan, and high/critical audit gate.
