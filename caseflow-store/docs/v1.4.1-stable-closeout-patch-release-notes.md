## What Changed

- Fixed compact book-card layout collisions that could make covers overlap
  labels, titles, prices, and edition-comparison content on homepage and book
  detail surfaces.
- Hardened the compact-card overlap verifier to cover homepage translated
  edition links, product edition-comparison cards, related book cards, content
  readability width, horizontal overflow, and right-edge clipping.
- Added authenticated customer order cancellation from account order history for
  cancellable pending/confirmed orders before payment and fulfillment reach
  terminal states.
- Kept customer order history account-scoped: signed-in customers can list and
  inspect their own orders, while cross-customer access remains denied.
- Clarified admin/staff order operations copy so `cancelled` is presented as
  rejected/cancelled for risky, invalid-contact, unavailable-stock, or other
  operationally unsafe orders.
- Verified staff/admin rejection through UI and persistence: staff can filter a
  risky order, set order/payment/shipping status to cancelled, save internal
  notes, and read the same terminal state back from the database.
- Adjusted the admin orders desktop breakpoint so the order list and detail
  panel do not squeeze status badges at standard 1440px desktop width.
- Shipped the post-release operations-login copy polish prepared during the
  v1.4 final QA pass.

## What Did Not Change

- No database schema migration or production catalog mutation was performed.
- No dependency was added.
- No real payment provider, shipping carrier, SMS/OTP provider, email
  verification provider, or external AI service was integrated.
- No commercial book-cover import, publisher metadata feed, fake review,
  fake sold-count, or fake urgency signal was added.
- Server-owned price, stock, VAT, shipping, payment-fee, promotion, total,
  role, order-ownership, and admin/staff authorization boundaries remain
  unchanged.
