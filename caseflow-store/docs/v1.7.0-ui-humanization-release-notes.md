## What Changed

- Reworked the public storefront away from generic SaaS-style symmetry and
  toward an editorial bookstore composition.
- Added a reading-table and spine-rail visual motif for the homepage.
- Removed public admin navigation from the customer-facing header while
  preserving the protected `/admin` route.
- Changed signed-in header copy to lead customers toward their account instead
  of exposing implementation state.
- Reduced heavy card/pill repetition in the homepage and catalog intro.
- Rewrote detail-page edition confidence copy into customer-facing language.
- Improved detail metadata wrapping for long ISBN/publisher values.
- Fixed the user-reported reading-path label overflow where `Step 2` and
  `Step 4` could wrap vertically inside compact product tiles.
- Fixed admin dashboard payment summaries so cancelled/rejected orders with old
  active payment states no longer appear as collectable pending payments.
- Normalized server-side admin/staff cancellation so open payment and shipping
  states become `cancelled` when an order is rejected/cancelled.
- Updated catalog and E2E verifiers to the current 500-edition baseline.

## Boundaries

- This release does not add a new database migration, payment provider,
  shipping carrier, email/SMS provider, commercial cover import, or external AI
  service.
- The 500 products remain edition variants across 50 works.
- QR payment remains a development/sandbox demo flow and is locked from
  production settlement.
- No fake sales figures, testimonials, reviews, ratings, or commercial claims
  were added.
