# CaseFlow Books v1.4 Release Readiness Report

- Date: 2026-07-19
- Task: `V14-T12 - Full Local Quality Gate And Release Readiness Report`
- Candidate: `v1.4.0` local release candidate
- Scope: real commerce copy, trust/policy surfaces, visual merchandising
  variety, checkout/customer/admin polish
- Result: locally release-ready; deployment, tag, and GitHub Release are not
  created by this task

## Summary

The `v1.4` candidate passes the local quality gate for a bounded real-commerce
and visual-merchandising upgrade. The storefront now has more varied
merchandising structures, customer-facing runtime copy avoids demo/internal
phrasing, policy pages are linked and route-safe, checkout reads like a
bookstore order flow, and admin surfaces read more like an operating console.

This report does not approve real payment processing, real carrier shipping,
SMS/OTP, external AI, commercial book-cover copying, or production data
migration. Those remain outside `ADR-0009` unless a later ADR/task explicitly
approves them.

## Changed Surfaces

- Homepage: category spine rail, editorial feature, reading path, translation
  pair, and deal strip layouts.
- Catalog: quick discovery links, active-filter summary, and multiple card
  structures tied to offer, translation, editorial, academic, trust, and
  standard catalog states.
- Product detail: commercial trust panels, edition identity, purchase proof
  context, and recommendation cards that preserve the compact-card overlap
  hotfix.
- Checkout: COD and bank transfer priority, policy links, customer-facing
  payment wording, and clearer VND/USD estimate language.
- Customer/account/tracking/assistant: account and tracking copy no longer
  exposes implementation language; public tracking shows a privacy guard; the
  assistant keeps bookstore-style guidance without creating orders from chat.
- Admin: dashboard, orders, catalog, inventory, and customers surfaces now use
  operations rails, dense status panels, mobile order cards, and admin-specific
  visual treatment.
- Policies/footer: contact, shipping, payment, returns, privacy, and terms
  pages are route-language-aware and included in footer, sitemap, and robots.

## Findings

No P0/P1 release-blocking issue remains open from the local gate.

One QA-tool issue was found and fixed during this task: the older
accessibility/mobile/performance verifier timed out while taking a full-page
homepage screenshot after the V14 homepage became longer and more image-heavy.
The runtime app did not fail an assertion. The verifier now disables animation,
keeps normal caret handling, and uses a 60-second screenshot timeout; the gate
passes after the fix.

## Accepted Non-Blockers

- Checkout remains a simulated bookstore order flow and does not submit real
  COD, bank transfer, MoMo, ZaloPay, VNPay, card, or bank credentials.
- Customer profile email and phone are required for checkout readiness, but the
  app still does not integrate a real email/SMS verification provider.
- Shipping, VAT, FX, and international payment-fee values remain configurable
  estimates, not legal/tax/carrier quotes.
- Public tracking is guarded by order code plus matching contact, but no
  dedicated rate limiter is implemented beyond platform/app boundaries.
- The active catalog still uses project-created CaseFlow cover assets. The
  user-provided Project Gutenberg, Standard Ebooks, and Vietnamese ebook/PDF
  sources are recorded as future source candidates, not approved runtime cover
  assets without provenance review.
- `npm audit` still reports two moderate PostCSS advisories inherited through
  `next@16.2.10`; `npm audit --audit-level=high` passes, and the suggested
  force fix is rejected because it proposes a breaking downgrade path.

## Release Recommendation

The `v1.4.0` candidate is ready for an explicit release task if the user wants
to deploy, smoke test production, commit, tag, push, and create a GitHub
Release. This `V14-T12` task intentionally stops before deployment or tagging
because `ADR-0009` requires separate release authorization.
