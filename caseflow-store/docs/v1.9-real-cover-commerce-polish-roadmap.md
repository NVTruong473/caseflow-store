# v1.9 Real Cover Commerce Polish Roadmap

- Status: Accepted
- Date: 2026-07-21
- Governing ADR: `docs/adr/0013-real-cover-and-commerce-homepage-upgrade.md`

## V19-T01 - Asset Metadata And Assistant Recovery Hardening

Acceptance criteria:

- Existing generated SVG cover metadata and generator/provenance markers are
  removed from public image files.
- Future generated cover scripts do not reintroduce embedded metadata blocks.
- The rule-based assistant answers all built-in suggested customer questions.
- Malformed catalog requests recover with similar search guidance.
- Admin/security/off-topic requests return the approved neutral fallback.

Verification:

- `node scripts/verify-public-asset-metadata.mjs`
- `ASSISTANT_UAT_BASE_URL=<local-url> npm exec -- tsx scripts/verify-assistant-customer-questions.ts`

## V19-T02 - Project Gutenberg Cover Asset Pipeline

Acceptance criteria:

- A reproducible script downloads accessible Project Gutenberg cover thumbnails
  into `public/images/books/gutenberg-covers`.
- Downloaded JPEGs are stored locally and stripped of metadata.
- A manifest records source URL, local path, checksum, checked timestamp, and
  use notes for each work.
- The script supports dry-run and apply modes.
- Apply mode updates only cover assets and edition cover references for works
  with confirmed local files.
- No storefront cover path points to a remote URL.

Verification:

- `npm exec -- tsx scripts/apply-v19-gutenberg-covers.ts --download`
- `npm exec -- tsx scripts/apply-v19-gutenberg-covers.ts --verify-local`
- If deploying/applying production data: run deploy first, then
  `npm exec -- tsx scripts/apply-v19-gutenberg-covers.ts --apply`

## V19-T03 - Homepage Retail Floor Redesign

Acceptance criteria:

- Homepage first viewport presents a real bookstore retail floor, not a generic
  centered SaaS/portfolio hero.
- Search, category entry points, live shelves, and featured covers are visible
  without exposing implementation-style copy.
- Layout uses deliberate asymmetry, local cover assets, and varied shelf
  structure.
- No fake metrics, fake reviews, or fake trust claims are introduced.
- Desktop and mobile layouts have no horizontal overflow or clipped cover/text
  collisions.

Verification:

- `V19_BASE_URL=<local-url> npm exec -- tsx scripts/verify-v19-commerce-homepage.ts`
- `npm run lint`
- `npm exec -- tsc --noEmit --pretty false`
- `npm run build`
- `git diff --check`

## V19-T04 - Optional Production Ship

Acceptance criteria:

- Code is committed after local checks pass.
