# CaseFlow Books v1.2 Editorial Metadata

Status: accepted for `V12-T08` on 2026-07-17.

## Outcome

`V12-T08` creates a public editorial metadata manifest for the canonical v1.2
catalog. It joins the accepted canonical catalog and 100-cover portfolio into a
single import-ready content shape for later migration and storefront work.

The accepted outputs are:

- `src/data/books/v1.2-editorial-metadata-manifest.json`
- `src/types/editorial-metadata.ts`
- `src/lib/validation/editorial-metadata.ts`
- `scripts/build-v12-editorial-metadata.ts`
- `scripts/verify-v12-editorial-metadata.ts`
- `.agent/artifacts/v12-t08/editorial-metadata-check.{json,md}`

## Content Shape

Each of the 100 edition records contains:

- localized display title and the public title for the edition language;
- authors, categories, work/edition pair references, and language;
- bilingual project-written summary;
- bilingual reason-to-read / merchandising note;
- primary cover path, bilingual alt text, and cover provenance id;
- display-safe optional facts only when a reviewed value exists;
- omitted optional fact keys when a value is unavailable;
- content-quality evidence for ADR-0007 blocking requirements.

The manifest intentionally omits unavailable optional facts rather than showing
`TBC`, `demo`, `placeholder`, `seed`, `debug`, `null`, or `undefined`.

The verifier checks exact edition coverage, cover coverage, bilingual
completeness, Vietnamese copy diacritics where meaningful, length ranges,
prohibited phrases, duplicate copy outside paired editions, internal pair
consistency, content-quality release readiness, and graceful optional-fact
omissions.

## Boundary

This task does not mutate Supabase, seed runtime rows, or change storefront
rendering. Runtime replacement belongs to the migration/import and UI tasks.

The public copy remains project-written. No publisher blurb, retailer copy,
customer review, protected excerpt, fabricated rating, sold count, or bestseller
claim is introduced.
