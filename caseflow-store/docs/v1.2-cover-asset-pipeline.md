# CaseFlow Books v1.2 Cover Asset Pipeline

Status: accepted for `V12-T06` on 2026-07-17.

## Outcome

`V12-T06` adds a deterministic cover-asset pilot for CaseFlow Books `v1.2`.
The pipeline does not replace the runtime catalog yet. It proves the asset
rules, metadata contract, visual smoke path, and rights boundary that must be
used before `V12-T07` produces all 100 covers.

The pilot outputs are:

- `src/data/books/v1.2-cover-pilot-manifest.json`
- `public/images/books/v12-pilot/*.svg`
- `scripts/build-v12-cover-pilot.ts`
- `scripts/verify-v12-cover-pipeline.ts`
- `.agent/artifacts/v12-t06/cover-pilot-contact-sheet.svg`
- `.agent/artifacts/v12-t06/cover-pilot-preview.html`
- `.agent/artifacts/v12-t06/cover-pipeline-check.{json,md}`
- `.agent/artifacts/v12-t06/cover-pilot-preview-{desktop,mobile}.png`

## Asset Rules

The accepted pilot uses project-created SVG artwork plus deterministic text
rendering. The script reads the canonical v1.2 manifest, chooses the correct
localized title and author data, wraps text in SVG, and writes local cover
files at `600x900` with a `2:3` aspect ratio.

The pilot deliberately rejects the shortcut that would make the catalog look
real faster but weaker as a portfolio artifact: copying commercial covers from
retailers, publishers, marketplaces, Google Images, Goodreads, Amazon, or
unreviewed search results.

Every pilot asset records:

- edition id, work id, pair id, slug, language, title, authors, and categories;
- art family and concept key;
- local path, dimensions, checksum, file size, and size budget;
- contrast ratios for title, author, and badge typography;
- bilingual alt text;
- approved project-created media provenance;
- an explicit note that no commercial cover reference was used.

## Pilot Coverage

The pilot contains 10 assets: five English/Vietnamese pairs and five visual
families.

| Work | Coverage reason |
| --- | --- |
| `Pride and Prejudice` | special-edition format, romance/classic, light artwork |
| `The Adventures of Sherlock Holmes` | long title, mystery/classic, dark artwork |
| `The War of the Worlds` | speculative/mystery categories, dark artwork |
| `Alice's Adventures in Wonderland` | young-reader fantasy, long localized title |
| `The Old Man and the Sea` | post-T05 replacement work, literary/sea concept |

The pilot covers both languages, long English and Vietnamese titles, light and
dark palettes, card-size rendering, and detail-size rendering.

## Rights Boundary

All pilot assets use the `project-created` rights basis. They do not use
external image references, commercial layout references, publisher marks,
retailer images, protected logos, or downloaded book-cover art.

If a future asset uses a licensed or public-domain source, it must carry a
complete provenance record with stable source URL, license, attribution rules,
review status, and replacement/takedown path. A failed rights check means the
asset is removed from primary rendering and replaced with a reviewed internal
or generated alternative.

## Size And Rendering Budget

The accepted pilot budget is:

| Rule | Value |
| --- | ---: |
| Output dimensions | `600x900` |
| Aspect ratio | `2:3` |
| Maximum file size per SVG | `55,000` bytes |
| Minimum text contrast | `4.5:1` |
| Generic placeholder primary count | not applicable until `V12-T07` |

The largest pilot SVG is under `4 KB`, well below the accepted budget. SVG is
acceptable for this pipeline because the artwork is project-created vector
asset output, not hand-drawn icons embedded into UI chrome.

## Verification

Run from `caseflow-store`:

```bash
npx tsx scripts/build-v12-cover-pilot.ts
npx tsx scripts/verify-v12-cover-pipeline.ts
npx tsc --noEmit
npm run lint
npm run build
git diff --check
```

The verifier checks schema validity, local file existence, checksums, dimensions,
file-size budget, absence of external image references, title/author rendering,
language coverage, long-title coverage, category coverage, light/dark artwork,
paired visual families, and desktop/mobile static preview rendering.

## Residual Risks

The pilot proves the pipeline and representative visual behavior; it does not
yet approve all 100 covers. `V12-T07` must expand this pipeline to the full
catalog, inspect contact sheets, reject weak or duplicate concepts, and confirm
that zero active editions still use the generic placeholder as their primary
catalog image.

The generated covers are illustrative CaseFlow assets, not real publisher
covers. Storefront copy and alt text must preserve that distinction unless the
project later acquires licensed or verified public-domain cover art.
