## Data Hygiene Correction

During T10 planning, two import risks were found before production write:

- Vietnamese retailer/product codes starting with `893` were being treated as
  ISBN-13 values. They are now omitted from `isbn13` because the database and
  storefront should reserve ISBN-13 for `978`/`979` Bookland identifiers.
- One Vietnamese `Wuthering Heights` source exposed publisher as `vh`. The
  source was replaced with a reviewed Minh Khai Books edition record for
  `Đồi Gió Hú (Tái Bản)`, and publisher display normalization now corrects
  obvious Unicode/casing issues without inventing a different publisher.

After regeneration:

- canonical verification passes with 100 editions and 72 valid ISBN-13 values;
- editorial verification passes with 334 display facts;
- supplemental checks report zero invalid ISBN display values and zero `vh` or
  malformed publisher display values.

## Migration Strategy

Use an expand-and-upsert strategy:

1. Capture provider backup/export evidence or an equivalent Supabase backup
   before production writes.
2. Capture count-only pre-migration state with
   `pre-migration-counts.sql` or the count-only script output.
3. Apply `0008_v12_catalog_merchandising.sql` in one transaction.
4. Upsert v1.2 catalog rows with deterministic IDs and conflict keys.
5. Deactivate retired `The Elements of Style` work/editions instead of
   deleting them.
6. Keep v1.1 placeholder and v1.1 seed data available until production smoke
   and rollback checks pass.

`V12-T11` was the first task allowed to write Supabase production data. It ran
only after explicit user confirmation.

## Upsert Idempotency

The data import in `V12-T11` should use these conflict keys:

| Table | Conflict key |
| --- | --- |
| `book_categories` | `id` |
| `book_authors` | `id` |
| `book_translators` | `id` |
| `book_publishers` | `id` |
| `book_cover_assets` | `id` |
| `book_works` | `id` |
| `book_work_authors` | `work_id,author_id` |
| `book_work_categories` | `work_id,category_id` |
| `book_editions` | `id` |
| `book_edition_translators` | `edition_id,translator_id` |
| `book_catalog_provenance_records` | `id` |
| `book_content_quality_checks` | `edition_id,requirement` |
| `book_catalog_compatibility` | `legacy_entity_type,legacy_id` |
| `book_merchandising_shelves` | `id` |
| `book_merchandising_shelf_items` | `shelf_id,edition_id` |

The import must not use table resets. Retired rows are made inactive with
targeted updates so historical order snapshots and public link compatibility
remain auditable.

## SQL Boundary

The migration is additive:

- adds v1.2 columns to `book_editions`;
- adds `book_catalog_provenance_records`;
- adds `book_content_quality_checks`;
- adds `book_catalog_compatibility`;
- adds `book_merchandising_shelves`;
- adds `book_merchandising_shelf_items`;
- enables RLS on the new tables;
- grants public read only where the storefront needs active compatibility or
  active merchandising data;
- keeps provenance/content-quality data service-role only.

The verifier rejects destructive SQL patterns such as table drops, truncation,
row deletes, order/profile table alteration, and dropped columns.

## Rollback

If the v1.2 import or smoke gate fails:

1. Disable merchandising shelves:

```sql
update public.book_merchandising_shelves
set is_active = false;
```

2. Deactivate v1.2-only catalog rows:

```sql
update public.book_editions
set is_active = false
where id in (
  '00000000-0000-4000-8000-000000007100',
  '00000000-0000-4000-8000-000000007200'
);

update public.book_works
set is_active = false
where id = '00000000-0000-4000-8000-000000007000';
```

3. Re-run the v1.1 seed path or restore the private pre-v1.2 catalog backup if
   production had manual catalog edits.
4. Do not delete customer, order, order item, promotion, inventory-adjustment,
   auth, phone catalog, or profile data.
5. Do not move or retag `v1.1.0`.
