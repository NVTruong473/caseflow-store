# Security Policy

## Supported Version

Only the latest immutable `template-v1.0.x` baseline is evaluated for shared
template security fixes. Each buyer repository must define its own supported
versions after customization.

## Report A Vulnerability

Report suspected vulnerabilities privately to the repository owner through
the established commercial channel or a private GitHub Security Advisory.
Never put credentials, personal data, buyer infrastructure details, or an
unpublished proof of concept in a public issue.

Include the affected template tag, reproducible steps, impact, and a minimal
safe proof of concept. Do not test against the public showroom database or a
buyer environment without written authorization.

Repository access does not include a bug bounty, incident-response service, or
guaranteed response time. Buyer-specific obligations require a written
agreement.
