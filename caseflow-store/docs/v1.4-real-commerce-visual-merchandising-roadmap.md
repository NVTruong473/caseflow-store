# CaseFlow Books v1.4 Real Commerce And Visual Merchandising Roadmap

- Status: released as `v1.4.0`
- Accepted date: 2026-07-19
- Governing ADR: `docs/adr/0009-real-commerce-visual-merchandising-upgrade.md`
- Scope: post-`v1.3.1` runtime polish and visual merchandising upgrade
- Release result: `v1.4.0` was deployed, smoke tested, tagged, pushed, and
  released only after explicit user approval in `V14-T13`.

## Goal

Make CaseFlow Books feel more like a small bookstore that is actively selling
and operating online, while keeping the project honest about payment,
licensing, data, and integration boundaries.

This roadmap combines two needs:

- commercial reality: remove customer-facing demo/internal language and present
  checkout, policies, trust, and operations as a believable bookstore flow;
- visual variety: replace the repeated bordered-card rhythm with structurally
  distinct merchandising surfaces, product card variants, and operational
  layouts.

## Non-Goals

- No real payment-provider integration.
- No SMS/OTP, email provider, shipping carrier, external AI, or new external
  service.
- No database schema migration or production data mutation unless a later task
  explicitly creates an approved migration plan.
- No fake review, fake sold count, fake bestseller, fake testimonial, or fake
  urgency.
- No copied commercial cover art, protected blurbs, protected excerpts, or
  marketplace images.
- No rewrite of the application, route tree, auth model, cart model, or API
  error contract.

## Release Gates

| Gate | Required Evidence |
|---|---|
| Planning gate | ADR accepted, roadmap accepted, task IDs and verification defined |
| Copy gate | Runtime no-demo-copy scan passes for customer-facing code |
| Visual gate | Homepage, catalog, detail, checkout, assistant, account/tracking, and admin screenshots at mobile and desktop |
| Commerce gate | Checkout and policy copy use customer-facing commercial wording without pretending to process real provider credentials |
| Security gate | Auth/access-control checks for anonymous, customer, staff, and admin still pass |
| Quality gate | TypeScript, ESLint, build, affected Playwright checks, visual QA, no horizontal overflow, no text overlap |
| Release gate | Optional deploy/tag/release only after explicit release task |
