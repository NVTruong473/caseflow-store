# CaseFlow Books v1.1 Safe Cover Asset Strategy

- Status: Accepted for `D24-T02`
- Date: 2026-07-16
- Related roadmap task: `D24-T02 - Create Safe Cover Asset Strategy`

## Decision

Use an internal placeholder cover for the Day 24 seed dataset until the project
has a licensed, generated, or fully internal cover pipeline.

The current placeholder is:

```text
/images/books/placeholders/book-cover-placeholder.svg
```

It is project-created, local, and not a hotlink to a commercial bookstore,
publisher, or review site.

## Why

Book cover art is a higher-risk asset than factual metadata. A real title can
be used as bibliographic data, but copying a commercial cover image can create
copyright and licensing risk. D24-T02 therefore favors a boring but honest
placeholder over visually attractive unlicensed covers.

## Rules

- Do not hotlink commercial cover images from bookstores, publishers, Google
  Images, Goodreads, Amazon, or marketplace pages.
- Do not copy cover files from publisher pages unless the license is explicit
  and recorded.
- Do not imply that a placeholder is the real cover of a specific edition.
- Use local placeholder, generated, internal, licensed, or public-domain assets
  only.
- Record source and license notes for every non-placeholder cover.
- Keep cover IDs stable so deterministic seeding can rerun safely.

## Current Mapping

All Day 24 editions map to one internal placeholder cover asset:

```text
coverAssetId: 00000000-0000-4000-8000-000000000050
path: /images/books/placeholders/book-cover-placeholder.svg
source: internal
```

This gives every edition a stable image reference without pretending to have
publisher-approved cover art.

## Missing Cover Behavior

If an edition has no `coverImageId`, the storefront should fall back to the
same placeholder asset. Future UI work should preserve fixed cover dimensions
so missing or slow-loading images do not resize product cards.

## Future Upgrade Path

Later tasks may replace placeholders with:

- Generated covers created specifically for this project.
- Internal typographic covers using the CaseFlow Books design system.
- Public-domain cover scans with documented source and license.
- Licensed assets stored locally with source notes.

The upgrade should be treated as a content/asset task, not a silent data edit.
