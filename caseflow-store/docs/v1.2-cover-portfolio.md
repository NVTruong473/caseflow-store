# CaseFlow Books v1.2 Cover Portfolio

Status: accepted for `V12-T07` on 2026-07-17.

## Outcome

`V12-T07` expands the accepted `V12-T06` cover pipeline from a representative
pilot to a complete 100-asset cover portfolio for the canonical v1.2 catalog.
At acceptance time it was an import input only. As of `V12-T11` through
`V12-T16`, the accepted assets are imported into Supabase, attached to the 100
active edition rows, rendered by homepage/catalog/detail surfaces, returned by
cart validation snapshots, and used by product social metadata/JSON-LD.

The accepted outputs are:

- `src/data/books/v1.2-cover-portfolio-manifest.json`
- `public/images/books/v12-covers/*.svg`
- `scripts/build-v12-cover-portfolio.ts`

- `.agent/artifacts/v12-t07/cover-portfolio-contact-sheet-all.svg`

art family stays stable for comparison.

## Rights And Source Boundary

All 100 assets are project-created SVG illustrations. The portfolio does not
use commercial cover art, publisher marks, bookstore images, marketplace
images, downloaded files, hotlinked images, copied layouts, or generated-image
references.

Every asset has:

- approved `cover-asset` provenance;
- `project-created` media rights basis;
- no external license requirement;
- bilingual alt text;
- deterministic title and author text from the canonical v1.2 manifest;
- checksum, dimensions, aspect ratio, file size, and contrast data.

These covers must be described as CaseFlow illustrative covers unless a future
task replaces an asset with licensed or verified public-domain cover art and
records the required provenance.

## Verification

Run from `caseflow-store`:

```bash
npx tsx scripts/build-v12-cover-portfolio.ts
npx tsx scripts/verify-v12-cover-portfolio.ts
npx tsx scripts/verify-v12-cover-pipeline.ts
npx tsc --noEmit
npm run lint
npm run build
git diff --check
```

The portfolio verifier checks exact edition coverage, language balance, pair
families, placeholder references, local files, checksums, dimensions, file-size
budget, external image references, protected mark patterns, duplicate hashes,
title/author rendering, source records, contact sheets, and representative
desktop/mobile preview rendering.

## Visual Review Notes

Representative desktop and mobile preview screenshots rendered without blank
images, horizontal overflow, or obvious clipped titles. The all-cover contact
sheet was also rendered to PNG for visual inspection.

The portfolio is intentionally systematic. It gives CaseFlow Books a credible
internal visual identity and avoids copyright shortcuts, but it is not claiming
publisher-designed cover art.

## Residual Risks

The active storefront no longer depends on the v1.1 placeholder as a primary
cover source. Placeholder rendering remains a defensive fallback for missing
future assets and an admin content-quality state. These covers are still
CaseFlow-created illustrative covers, not publisher-designed cover art or a
licensed commercial image feed.

Current runtime integration is verified by `V12-T16`; future media changes
still need explicit licensing/provenance, upload, processing, CDN, alt-text, and
takedown handling.
